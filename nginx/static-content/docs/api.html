API docs

Where full URLs are provided in responses they will be rendered as if service is running on 'https://voltjunkie.ru/'.

Open Endpoints

Open endpoints require no authentication.

  • Authenticate : POST /api/auth/jwt/authenticate?username={value}&password={value}
  • Register : POST /api/auth/jwt/register?username={value}&password={value}&email={value}
  • Get public key used in jwt signing : GET /api/auth/jwt/publicKey

Endpoints that require Authentication

Closed endpoints require a valid Token to be included in the header of the request. A Token can be acquired from the Login view above

User-service related

Each endpoint manipulates or displays information related to the users. Token must be provided with the request:

  • Create user : POST /api/users?username={value}&email={value}&password={value}
  • Show user : GET /api/users/{user_id}
  • Show all users : GET /api/users
  • Update info : PATCH /api/users/{user_id}?username={value}&email={value}&isEmailConfirmed={value}&isDeleted={value}&password={value}&role={value}
  • Delete user : DELETE /api/users/{user_id}
  • Confirm email : GET /api/users/email/{email_id}

Authentication-service related

Endoints with public RSA keys

  • Refresh token : POST /api/auth/jwt/refresh

Recipes-service related

Each endpoint manipulates or displays information related to the recipes. Token must be provided with the request:

  • Get all recipes : GET /api/recipes
  • Get recipe : GET /api/recipes/{recipe_id}
  • Create recipe : POST /api/recipes
  • Delete recipe : DELETE /api/recipes/{recipe_id}
